# 欣威视通遥控器 APK 设计规格

日期：2026-06-01
产品名：欣威视通遥控器
目标平台：Android 7.0 到 Android 16

## 背景

本 APK 用于把实体终端遥控器做成 Android 应用。参考资料包括：

- `reference/terminal_remote_manual_v2.docx`：终端遥控器使用说明。
- `reference/remote_key_codes.pdf`：标准遥控器键码说明，用户码为 `0x00FF`。
- 用户提供的产品确认书截图：遥控器外观、按键位置、颜色体系和键码布局。

## 产品目标

从高级产品经理角度，本产品应优先满足三类场景：

1. 客户交付：应用名称、首次启动说明、主界面视觉需要清楚、正式、可信。
2. 售后支持：用户可以按实体遥控器习惯快速找到按键，减少培训成本。
3. 工程调试：高级用户可以开启键码显示，便于确认按键码值和排查问题。
4. 国际化交付：支持中文和英文两个版本，默认随系统语言显示，也允许用户在应用内主动切换。

从小白用户角度，本产品应做到：

1. 打开就能用，不需要理解键码或协议。
2. 常用按键位置接近真实遥控器，可对照实体图操作。
3. 每次按键有明确反馈，如按钮按压态、Toast 或状态提示。
4. 首次启动有简短说明，避免长篇文档式教学。

## 视觉与交互方向

已确认采用「精致仿真专业版」：

- 主界面保留真实遥控器布局。
- 整体为深色遥控器面板，具备轻量材质感。
- 电源键使用红色，信号源键使用绿色，导航区使用蓝紫色，普通功能键使用浅灰色。
- 顶部显示产品名“欣威视通遥控器”和用户码 `00FF`。
- 重要功能使用中文标签，必要时保留英文缩写，例如 `HDMI`、`VGA`、`YPbPr`、`AV`。
- 默认不显示键码，设置中可开启“显示键码”调试模式。
- 文案支持中文和英文；用户选择“跟随系统”时，根据系统语言自动显示对应版本。

主界面不做营销式首页，打开应用后直接进入遥控器。

## 语言策略

应用支持三种语言模式：

1. 跟随系统：默认模式。系统语言为中文时显示中文；其他语言默认显示英文。
2. 简体中文：用户手动选择后固定显示中文。
3. English：用户手动选择后固定显示英文。

语言切换入口放在设置页，切换后立即生效。首次启动说明、主界面按钮说明、设置页、Toast/状态提示、错误提示和版本信息均使用同一套语言资源。

应用名称建议：

- 中文：欣威视通遥控器
- 英文：SIGNWAY Remote Control

按钮标签策略：

- 强品牌或通用缩写保持一致，例如 `HDMI`、`VGA`、`YPbPr`、`AV`、`F1` 到 `F4`。
- 常用功能根据语言显示，例如“电源 / Power”、“信号源 / Source”、“音量+ / Volume+”。
- 调试模式下键码不翻译，统一显示 `0x45` 这类十六进制值。

## 首次启动说明

首次启动显示 3 页引导，用户可跳过：

1. 基础控制：介绍电源、方向、确认、返回、主页、播放/暂停。
2. 菜单与信号源：介绍 `MENU`、`SETUP`、`SOURCE`、`HDMI`、`VGA`、`YPbPr`、`AV`。
3. 调试与反馈：说明每次按键会显示反馈；高级用户可在设置中开启键码显示。

引导只在首次启动展示。用户之后可从设置页再次查看。

首次启动说明也跟随当前语言模式：

- 默认跟随系统语言。
- 如果用户在引导页切换语言，后续页面和应用设置同步使用该语言模式。

首版实现：

- 完整 Android 工程结构。
- 应用名称：欣威视通遥控器。

## 架构

建议模块：

- `MainActivity`：应用入口，承载传统原生 View UI。
- `RemoteControlScreen`：遥控器主界面。
- `IntroScreen` 或 `IntroDialog`：首次启动功能说明。
- `SettingsScreen`：设置、调试模式、重新查看说明。
- `RemoteKey`：按键模型，包含标签、键码、分组、视觉类型和说明。
- `RemoteCommandSender`：发送接口。
- `LoggingRemoteCommandSender`：首版占位实现，用日志和 UI 反馈记录发送动作。
- `PreferencesRepository`：保存首次启动状态、调试显示状态和语言模式。
- `LanguageRepository` 或等价配置层：读取系统语言、用户语言偏好，并向 UI 提供当前语言。

数据流：

1. 用户点击按钮。
2. UI 找到对应 `RemoteKey`。
3. ViewModel 调用 `RemoteCommandSender.send(remoteKey)`。
4. 发送层返回结果。
5. UI 显示反馈，例如“已发送：电源 0x45”。

语言数据流：

1. 启动时读取用户语言模式。
2. 如果模式为“跟随系统”，根据当前系统 Locale 选择中文或英文。
3. 如果用户手动选择语言，将选择保存到本地设置。
4. UI 根据当前语言资源刷新显示。

## 兼容性策略

- `minSdk`：24，对应 Android 7.0。
- 首版本地构建使用 `compileSdk` / `targetSdk` 34，运行兼容 Android 16；当构建工具升级到完整支持 Android 16/API 36 后，再提升到 36。
- 为保证 APK 体积更小、启动更快、可靠稳定，首版不采用 Jetpack Compose、AppCompat 或 Material 组件库；使用平台原生 Activity、View 和资源文件实现。
- 避免 Android 8 以后才有的 API 直接调用；如必须使用，添加版本判断。
- 使用 `SharedPreferences` 保存简单设置，保证 Android 7.0 可用。
- 语言切换不依赖 Android 13 的 App Languages API；首版使用应用内语言状态管理，保证 Android 7.0 到 Android 16 一致可用。
- 如果后续接入 Android 13+ 系统级应用语言设置，也必须保留应用内切换入口。
- UI 尺寸适配手机和常见平板屏幕：
  - 小屏幕可纵向滚动。
  - 遥控器面板使用最大宽度约束。
  - 按钮有稳定尺寸，避免文字挤压和布局跳动。
- 不依赖网络权限，除非后续新增网络控制协议。

## 测试策略

单元测试：

- 键码表完整性：所有资料中的键都有映射。
- 键码格式：所有键码在 `0x00` 到 `0xFF` 范围内。
- 调试模式：打开后按钮显示键码，关闭后隐藏。
- 首次启动状态：首次显示说明，确认后不再自动显示。
- 语言模式：默认跟随系统；中文系统显示中文；非中文系统显示英文；手动选择后不再随系统变化。

UI 或截图验证：

- 主界面在窄屏和宽屏下不重叠、不截断。
- 关键按钮位置接近确认图。
- 首次启动说明可跳过、可完成。
- 中文和英文两套界面都无文字截断，尤其是 `Play/Pause`、`Volume+`、`Reopen Guide` 等较长英文。

手动验证：

- Android 7.0 到 Android 16 设备或模拟器安装启动。
- 点击全部按键有反馈。
- 设置页能开启键码显示并重新查看说明。
- 设置页可在“跟随系统 / 简体中文 / English”之间切换，切换后界面立即更新。

## 风险与约束

- 当前缺少真实发送协议。首版必须把发送层抽象出来，避免 UI 和后续硬件协议耦合。
- 若最终需要红外、串口、网络或 ADB 控制，应补充协议文档后再实现发送层。
- Android 7.0 兼容会限制部分现代 API 使用，需要保守配置依赖版本。
- 真实遥控器按键较多，小屏手机上需要平衡美观与可点击尺寸。
